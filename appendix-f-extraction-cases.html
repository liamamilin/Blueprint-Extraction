32  Appendix F — Five Complete Extraction Cases (with Evidence Trace)

This appendix presents five representative blueprint extraction cases.
Each case demonstrates how repository signals, evidence objects, and mapping rules combine to produce a structured blueprint kernel.

The examples are intentionally simplified but preserve the full extraction logic:

signal → evidence → mapping → kernel slot → blueprint element

Each case includes an evidence trace, showing how architectural conclusions are justified.

38 Summary

These five cases illustrate the full blueprint extraction process:

repository artifact
        ↓
signal detection
        ↓
evidence object
        ↓
ontology mapping
        ↓
kernel slot filling
        ↓
blueprint element

Across different system types—CLI tools, API services, data pipelines, event-driven systems, and agents—the extraction pipeline follows the same disciplined workflow.

Each architectural element remains traceable to repository evidence, ensuring that blueprint reconstruction remains auditable and reproducible.
