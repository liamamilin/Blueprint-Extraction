33  Appendix G — Extraction Maturity Level (EML) Score Sheet

The Extraction Maturity Level (EML) score sheet provides a standardized method for evaluating the quality and reliability of blueprint extraction results.

While earlier sections defined EML conceptually, the score sheet operationalizes those levels using measurable criteria. It allows extraction pipelines, reviewers, and dataset maintainers to assess the maturity of individual blueprint packs and monitor the overall quality of blueprint repositories.

The scoring framework evaluates several dimensions of extraction quality, including structural coverage, evidence traceability, confidence distribution, and review status.

33.1 EML Overview

Extraction Maturity Levels represent progressively higher levels of blueprint reconstruction quality.

EML 0 — No structured extraction
EML 1 — Signal detection only
EML 2 — Partial kernel reconstruction
EML 3 — Evidence-backed blueprint draft
EML 4 — Reviewed reference blueprint

Each level corresponds to specific operational criteria that must be satisfied before the blueprint can be classified at that level.

33.2 Scoring Dimensions

The EML score sheet evaluates blueprint extraction across four primary dimensions:

  • Kernel coverage

  • Evidence traceability

  • Structural consistency

  • Review status

Each dimension contributes to determining the final maturity level.

33.3 Kernel Coverage

Kernel coverage evaluates whether the six blueprint kernel slots have been populated with meaningful architectural elements.

Goal
Capabilities
Components
Interfaces
Data
Workflows

Coverage is evaluated as follows:

Coverage Level Description
None No kernel elements reconstructed
Partial Some kernel slots populated
Substantial Most slots populated with valid entities
Complete All slots populated or explicitly marked unknown

Coverage indicates how completely the system architecture has been reconstructed.

33.4 Evidence Traceability

Evidence traceability measures whether architectural elements are supported by traceable evidence objects.

Evidence Ratio Interpretation
< 30% Mostly speculative extraction
30–60% Mixed evidence and inference
60–85% Strong evidence support
> 85% Highly traceable architecture

Higher evidence ratios indicate stronger auditability.

Evidence traceability is essential for determining whether the blueprint can be trusted for architectural analysis.

33.5 Structural Consistency

Structural consistency evaluates whether the extracted blueprint forms a coherent architecture graph.

Checks include:

  • valid references between entities

  • capabilities connected to interfaces or workflows

  • components participating in interactions

  • data entities owned by components

  • workflows referencing valid execution units

Consistency levels may be classified as:

Level Description
Low Frequent anomalies or broken relationships
Moderate Minor inconsistencies present
High Architecture graph internally coherent

Consistency indicates the structural reliability of the blueprint.

33.6 Review Status

Review status reflects whether the blueprint has undergone human validation.

Status Meaning
None No human review performed
Partial Limited review of critical elements
Complete Full review according to protocol

Human review significantly increases blueprint reliability, particularly for inferred elements.

33.7 EML Score Table

The following table summarizes the criteria associated with each maturity level.

EML Level Kernel Coverage Evidence Ratio Consistency Review Status
EML 0 None N/A N/A None
EML 1 Minimal signals detected <30% Unknown None
EML 2 Partial kernel reconstruction 30–60% Moderate None
EML 3 Substantial kernel coverage 60–85% High Optional
EML 4 Complete kernel representation >85% High Complete

This table provides a quick classification method for evaluating blueprint maturity.

33.8 Scoring Template

A blueprint pack may include an EML score entry similar to the following:

eml_score:
  level: EML3
  kernel_coverage: substantial
  evidence_ratio: 0.72
  structural_consistency: high
  review_status: partial

This structured representation allows automated systems to track extraction maturity across blueprint datasets.

33.9 Dataset-Level Usage

At scale, EML scores provide insight into the overall quality of blueprint mining pipelines.

Examples of dataset-level metrics include:

  • distribution of blueprints across EML levels

  • average evidence ratio across repositories

  • proportion of reviewed blueprints

  • extraction maturity growth over time

These metrics help pipeline operators identify areas where extraction methods require improvement.

33.10 Purpose of the EML Framework

The EML score sheet transforms blueprint extraction from a qualitative process into a measurable engineering discipline.

It enables consistent evaluation of blueprint quality, supports large-scale architecture mining pipelines, and ensures that blueprint repositories maintain clear distinctions between preliminary extraction results and verified architectural knowledge.

By standardizing extraction maturity assessment, the EML framework supports the long-term goal of building reliable architecture datasets and architecture intelligence systems.
